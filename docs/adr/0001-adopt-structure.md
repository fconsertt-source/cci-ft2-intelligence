# ADR 0001 — اعتماد حالة المشروع والهيكل المرجعي

تاريخ: 2026-01-31

الحالة: مقبول

المراجع: `docs/ProjectState.md`, `docs/architecture/ENGINEERING_CHARTER.md`

## السياق

بناءً على ميثاق الهندسة، نحتاج إلى توثيق الوضع الحالي للشجرة النمطية للمشروع وضمان إمكانية تتبع التغييرات البنيوية مستقبلاً.

يوجد كود ووحدات خارج `src/` (مثل `ft2_reader/` و `ingestion/`) وملفات مساعدة في `scripts/`، ما قد يعيق تطبيق فصل الطبقات الصارم بدون توثيق ونقل منظم.

## القرار

1. اعتماد `docs/ProjectState.md` كمرجعية أولية لحالة المشروع ولهيكله المنطبق الآن.
2. إنشاء مجلد ADRs في `docs/adr/` وكل قرار بنيوي مستقبلي يجب أن يُسجل كملف ADR جديد بصيغة نصية مرتبة بأرقام متزايدة.
3. اعتبار الانحرافات الموثقة في `docs/ProjectState.md` كقضايا عمل (work items) للتخطيط والجدولة، وعدم البدء بأي refactor قبل أخذ موافقة ADR عند الحاجة.

## العواقب

- تحسين الشفافية والتتبع للتغييرات البنيوية.
- السماح بمراجعات مُسجلة قبل ترحيل المجلدات أو تغيير الواجهات.
